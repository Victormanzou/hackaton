portafolio

Algoritmo

Conjunto ordenado de operaciones sistemáticas que permiten hacer un cálculo y hallar la solución de un tipo de problemas.

Complejidad algorítmica

Estudio teórico de la cantidad de recursos computacionales (como tiempo y espacio) necesarios para ejecutar un algoritmo.

Eficiencia algorítmica

Medida de qué tan bien utiliza un algoritmo los recursos disponibles para realizar una tarea determinada.

Recursión

Técnica en la que una función se llama a sí misma para resolver subproblemas más pequeños.

Orden de complejidad

Representación del crecimiento de la cantidad de recursos (generalmente tiempo) necesarios para ejecutar un algoritmo en función del tamaño de la entrada.

Big O notation

Notación utilizada para describir el rendimiento o la complejidad temporal de un algoritmo en términos de la entrada.

Estructuras de datos

Formas de organizar y almacenar datos para que puedan ser utilizados eficientemente, como arrays, listas enlazadas, árboles, etc.

Programación dinámica

Técnica de optimización de algoritmos que involucra dividir un problema en subproblemas más pequeños y almacenar las soluciones para evitar recálculos.

Árbol de decisión

Estructura de datos utilizada en algoritmos de búsqueda y optimización que divide un conjunto de datos en subconjuntos más pequeños basados en ciertos criterios.

Heurística

Método que utiliza reglas generales y experiencia previa para encontrar soluciones aproximadas a problemas difíciles o complejos.

Lenguaje de programación

Conjunto de reglas sintácticas y semánticas utilizadas para escribir programas informáticos.

Iteración

Repetición de un conjunto de instrucciones para alcanzar una solución, a menudo utilizando bucles.

Compilador

Programa que traduce código escrito en un lenguaje de programación a un lenguaje de máquina que una computadora puede ejecutar

Depuración(debugging)

Proceso de identificación y corrección de errores en un programa de software.

Optimización

Proceso de mejorar el rendimiento y la eficiencia de un programa o algoritmo sin cambiar su funcionalidad.

Variable

Elemento fundamental en programación que almacena datos y cuyo valor puede cambiar durante la ejecución del programa.

Algoritmo de búsqueda

Método utilizado para encontrar un elemento particular dentro de un conjunto de datos, como búsqueda lineal o búsqueda binaria.

Algoritmo de ordenamiento

Procedimiento para organizar elementos en una secuencia ordenada, como el algoritmo de ordenamiento burbuja, el de selección o el quicksort.

Interfaz de programación de aplicaciones (API)

Conjunto de protocolos y herramientas que permiten a diferentes componentes de software interactuar entre sí.

Paradigma de programación

Enfoque o estilo de escribir programas, como programación orientada a objetos, programación funcional, etc.

Ámbito (scope)

Contexto en el que una variable o función es válida y accesible dentro de un programa.

Herencia

Concepto en programación orientada a objetos donde una clase puede heredar atributos y métodos de otra clase.

Polimorfismo

Capacidad de objetos de diferentes clases de responder al mismo mensaje de manera diferente.

Abstracción

Proceso de identificar las características esenciales de un objeto o entidad para modelarlo de manera simplificada en un programa.

Encapsulamiento

Principio de programación orientada a objetos que agrupa datos y métodos que operan en esos datos en una unidad cohesiva.

Algoritmo voraz (greedy algorithm)

Estrategia de resolución de problemas que consiste en tomar la decisión óptima en cada etapa local con la esperanza de encontrar una solución global óptima.

Algoritmo de vuelta atrás (backtracking)

Técnica de búsqueda sistemática para encontrar todas las soluciones posibles a un problema mediante la construcción de soluciones paso a paso, retrocediendo cuando se encuentra una solución que no cumple con los criterios del problema.

Algoritmo de ramificación y poda (branch and bound)

Estrategia de optimización para resolver problemas de optimización combinatoria mediante la exploración sistemática de todas las posibles soluciones y eliminación de aquellas que no pueden conducir a una solución óptima
